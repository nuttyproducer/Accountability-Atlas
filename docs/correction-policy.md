# Correction Policy

## Purpose

Corrections protect trust. Accountability Atlas should make it easy to report errors and should correct them quickly and transparently.

---

## What can be corrected

Corrections may involve:

- factual error;
- wrong date;
- wrong location;
- wrong name;
- outdated legal status;
- broken source link;
- mistranslation;
- misleading summary;
- incorrect verification label;
- unsafe personal information;
- copyright issue;
- false partnership implication;
- duplicated item;
- disputed source;
- accessibility issue.

---

## How to request a correction

Use the GitHub source correction issue template, or future correction form when available.

A useful correction request includes:

- page or item URL;
- what appears incorrect;
- suggested correction;
- source supporting the correction;
- urgency level;
- safety/privacy concern if any.

---

## Correction priority

### Urgent

- doxing/private data;
- dangerous location exposure;
- false allegation against a private person;
- security issue;
- legal status badly misstated;
- false partnership claim.

Target: immediate review.

### High

- material factual error;
- wrong casualty figure;
- wrong source attribution;
- misleading legal wording;
- mistranslation changing meaning.

Target: prompt review.

### Normal

- typo;
- formatting;
- outdated link;
- minor wording improvement.

Target: normal review cycle.

---

## Correction workflow

```text
correction received
→ triage urgency
→ check sources
→ ask reviewer if needed
→ update content
→ add correction note if material
→ close correction request
→ record in correction history
```

---

## Public correction notes

Material corrections should be visible on the affected page or item.

Example:

> Correction, 2026-07-09: This entry previously described the proceeding as a final judgment. It has been corrected to “provisional measures order.”

Minor typos do not require public correction notes.

---

## Retractions

If an item is unreliable, unsafe, or false, it may be retracted.

Retraction note should include:

- what was removed;
- why;
- date;
- whether replacement information exists.

Avoid republishing harmful details in the retraction note.

---

## Disputes

If sources disagree and the item remains relevant:

- mark as disputed;
- show source disagreement;
- avoid definitive wording;
- update if stronger evidence emerges.
