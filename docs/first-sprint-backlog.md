---

## Do not start in sprint 1

- witness submissions;
- live maps;
- scraping;
- user accounts;
- donation flows;
- AI translation;
- live analytics dashboards;
- automated mass email system;
- sensitive media handling.

---

## Sprint success criteria

At the end of sprint 1:

- a new contributor knows how to help;
- maintainers know what is safe and unsafe;
- project tone is clear;
- public call for collaborators can be posted;
- first small tasks are ready;
- no unsafe feature has been started.
